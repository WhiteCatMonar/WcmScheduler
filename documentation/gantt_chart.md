# ガントチャート仕様

このドキュメントは、WcmScheduler のガントチャート機能仕様を整理するための資料です。
プロジェクトスケジュール機能とチームスケジュール機能で共通利用するガントチャート資産を前提とします。

ここに記載する仕様は、実装状況に応じて更新します。

## 概要

ガントチャートは、タスク編集機能で作成したタスク情報をもとに、プロジェクトまたはチーム全体の予定を時系列で表示する機能です。

主に以下の情報を参照します。

- タスク名
- タスク間接続
- 担当者
- 作業協力者
- 作業見積時間
- 開始日時
- 終了日時
- 中断期間
- メンバーのプロジェクト参加期間
- メンバーの作業可能時間

ガントチャート機能は、以下の2つの画面から利用します。

- プロジェクトスケジュール
- チームスケジュール

## 共通資産

プロジェクトスケジュールとチームスケジュールは、共通のガントチャート部品を利用します。

想定する主な構成要素は以下です。

| 名称 | 役割 |
| --- | --- |
| `GanttChartControl` | ガントチャート表示用Control |
| `GanttChartViewModel` | ガントチャート全体の状態管理 |
| `GanttTaskItemViewModel` | ガントチャート上に表示するタスク1件分の状態管理 |
| `GanttTimelineViewModel` | 時間軸、表示範囲、日付ヘッダーの状態管理 |
| `GanttChartService` | 予定期間算定、依存関係解決、表示用データ生成 |

共通資産はViewModel中心で設計し、特定の画面やタスク編集UIに依存しすぎない構造とします。

## 表示粒度

表示上の時間軸最小粒度は日単位とします。

日時情報自体は分単位で保持しますが、ガントチャート上の目盛り、日付ヘッダー、横方向の表示単位は日単位です。

## プロジェクトスケジュール

プロジェクトスケジュールは、1つのプロジェクト内のタスクを表示するガントチャートです。

配置は、プロジェクトタブ内で「タスク編集」タブと並列にする想定です。

主な仕様は以下です。

- 同一プロジェクト内のタスクを表示する
- タスク編集タブのノード情報を参照する
- タスク間接続を依存関係として表示する
- タスク名、担当者、開始日時、終了日時を表示する
- ガントチャート上でタスク期間を編集できる
- ガントチャート上の中断期間も編集できる
- ガントチャート上の編集はUndo/Redo対象とする
- タスク編集タブの変更をガントチャートへ反映する
- ガントチャートの変更をタスク編集タブへ反映する

## チームスケジュール

チームスケジュールは、複数プロジェクトのタスクを横断表示するガントチャートです。

配置は、アプリケーション直下で「チーム内プロジェクト」「チーム設定」と並列にする想定です。

主な仕様は以下です。

- すべてのプロジェクトのタスクを同一表示領域に並列表示する
- プロジェクトを跨いだ依存関係解決は行わない
- プロジェクト別フィルタを検討する
- 担当者別フィルタを検討する
- 作業協力者別フィルタを検討する
- 並び替えを検討する
- プロジェクトスケジュールと同じガントチャート共通資産を利用する

## 予定期間算定

ガントチャート上に表示するタスク予定期間は、タスクの日時情報、作業見積時間、依存関係、担当者の作業可能時間から算定します。

### 基本ルール

| 条件 | 予定期間 |
| --- | --- |
| 開始日時と終了日時が両方設定済み | 開始日時から終了日時まで |
| 開始日時のみ設定済み | 開始日時と作業見積時間から算定 |
| 終了日時のみ設定済み | 作業見積時間から開始位置を逆算 |
| 開始日時と終了日時が未設定 | 前段タスクの最遅終了日時と作業見積時間から算定 |
| 作業見積時間が未設定または `0min` | 描画しない |

開始日時と終了日時が両方設定されている場合、作業見積時間は予定期間算定に使用しません。

### 前段タスク

開始日時と終了日時が未設定のタスクは、Inputに接続されている前段タスクを参照します。

前段タスクが複数ある場合は、予定終了日時が最も遅いタスクの終了日時を基準にします。

前段タスクが存在せず、開始日時も未設定の場合は、現在日付を基準に描画します。

### 担当者の作業可能時間

作業見積時間から予定期間を算定する場合、担当者の作業可能時間を参照します。

参照する情報は以下です。

- メンバーの曜日別デフォルト作業可能時間
- プロジェクト別参加期間
- プロジェクト別、メンバー別、日付別作業可能時間上書き

日付別作業可能時間が未入力の場合は、曜日別デフォルト作業可能時間を使用します。

作業可能時間が `0min` の日は、実質的な中断日として扱います。

### プロジェクト参加期間

担当者がプロジェクト参加期間外の日付では、そのプロジェクトの作業可能時間を持たないものとして扱います。

参加期間外に予定期間がかかる場合は、対象区間を網掛け表示などで視覚的に識別します。

## 中断期間

タスクには任意数の中断期間を設定できます。

ガントチャートでは、タスクの予定期間と中断期間が重なる場合に、中断区間を視覚的に表示します。

主な仕様は以下です。

- タスク予定期間と重なる中断期間のみ表示する
- タスク予定期間と重ならない中断期間は表示しない
- 重複または連続する中断期間は、算定前に正規化する
- 中断期間はグレーアウトした線または区間として表示する
- 中断期間はガントチャート上で編集可能とする

## 依存関係表示

タスク編集タブ上の接続線は、ガントチャート上ではタスク間依存関係として表示します。

主な仕様は以下です。

- OutputからInputへの接続を依存関係として扱う
- 依存元タスクから依存先タスクへ線を表示する
- プロジェクトスケジュールでは同一プロジェクト内の依存関係を表示する
- チームスケジュールではプロジェクト内の依存関係のみ表示する
- プロジェクトを跨いだ依存関係解決は行わない

## 編集

ガントチャート上では、タスク期間と中断期間を直接編集できるようにします。

主な仕様は以下です。

- タスクバーの移動で開始日時と終了日時を変更する
- タスクバー端の操作で開始日時または終了日時を変更する
- 中断期間をガントチャート上で編集する
- 編集結果はタスク詳細情報へ反映する
- 編集はUndo/Redo対象とする
- タスク詳細側の変更もガントチャートへ反映する

## ステータス表示

タスクステータスはタスク編集機能と同じ自動算出結果を使用します。

対象ステータスは以下です。

- `Ready`
- `Pending`
- `InProgress`
- `Done`

ガントチャート上では、ノードのステータス色と同じ色で表現します。

ステータスはフィルタ条件として利用することも検討します。

## 警告表示

ガントチャート上で警告が必要なタスクは、視覚的に識別できるようにします。

現時点の警告表示方針は以下です。

| 条件 | 表示 |
| --- | --- |
| 終了日時のみ設定されている | タスク名を警告色にする |
| 参加期間外に予定期間がかかる | 対象区間を網掛け表示などで示す |

## テーマ

ガントチャートはテーマ設定に対応します。

追加候補のテーマ項目は以下です。

- ガントチャート背景色
- 時間軸色
- グリッド線色
- タスクバー通常色
- タスクバー選択色
- タスクバー文字色
- 現在日時ライン色
- プロジェクト区切り色
- 中断期間表示色
- 依存関係線色
- 警告表示色
- 参加期間外区間表示色

## 保存データ

初期実装では、ガントチャートはタスク情報とメンバー情報から再生成できる表示機能として扱います。
そのため、現時点ではガントチャート表示状態を保存しません。

ビューのカスタマイズ機能などが追加された場合は、`app-settings.json` への保存を検討します。

将来的な保存候補は以下です。

- 表示開始日
- 表示終了日
- ズーム状態
- スクロール位置
- フィルタ状態
- 並び替え状態
- 折りたたみ状態
