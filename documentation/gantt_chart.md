# ガントチャート仕様

このドキュメントは、WcmScheduler のガントチャート機能仕様を整理するための資料です。
プロジェクトスケジュール機能とチームスケジュール機能で共通利用するガントチャート資産を前提とします。

ここに記載する仕様は、実装状況に応じて更新します。

## 概要

ガントチャートは、タスク編集機能で作成したタスク情報をもとに、プロジェクトまたはチーム全体の予定を時系列で表示する機能です。

主に以下の情報を参照します。

- タスク名
- タスク間接続
- 担当者
- 作業協力者
- 作業見積時間
- 開始日時
- 終了日時
- 中断期間
- メンバーのプロジェクト参加期間
- メンバーの作業可能時間

ガントチャート機能は、以下の2つの画面から利用します。

- プロジェクトスケジュール
- チームスケジュール

プロジェクトスケジュールは、プロジェクト内のタスク編集画面に統合された中央編集領域タブとして表示します。
タスク詳細編集と履歴操作は、依存関係編集と共通のサイドバーから行います。

## 共通資産

プロジェクトスケジュールとチームスケジュールは、共通のガントチャート部品を利用します。

想定する主な構成要素は以下です。

| 名称 | 役割 |
| --- | --- |
| `GanttChartControl` | ガントチャート表示用Control |
| `GanttChartViewModel` | ガントチャート全体の状態管理 |
| `GanttTaskItemViewModel` | ガントチャート上に表示するタスク1件分の状態管理 |
| `GanttTimelineViewModel` | 時間軸、表示範囲、日付ヘッダーの状態管理 |
| `GanttChartService` | 予定期間算定、依存関係解決、表示用データ生成 |

共通資産はViewModel中心で設計し、特定の画面やタスク編集UIに依存しすぎない構造とします。

## 表示粒度

表示上の時間軸最小粒度は日単位とします。

日時情報自体は分単位で保持しますが、ガントチャート上の目盛り、日付ヘッダー、横方向の表示単位は日単位です。

## 表示操作

ガントチャートは、左側のタスク名列、担当者列を固定表示し、右側のチャート領域のみをスクロール対象とします。

縦横スクロールは、チャート側で表示オフセットを管理します。
これにより、固定列とチャート本体の行位置を一致させます。

主な操作は以下です。

| 操作 | 動作 |
| --- | --- |
| マウスホイール | 縦スクロール |
| `Shift` + マウスホイール | 横スクロール |
| `Ctrl` + マウスホイール | 横方向拡縮 |

横方向拡縮は、日単位の表示幅を変更します。
縦方向の行高さは、タスク一覧との対応を保つため初期実装では固定とします。

## フィルタ・並び替え

ガントチャートでは、表示対象タスクを絞り込み、必要な順序で確認できるようにします。

フィルタ・並び替えは、プロジェクトスケジュールとチームスケジュールの共通機能として扱います。
実装はガントチャート関連ViewModelに集約し、特定画面に依存しない構造とします。

### フィルタ対象

初期実装で対応するフィルタ対象は以下です。

- 担当者
- 作業協力者
- ステータス

複数のフィルタ条件を指定した場合、条件間はAND条件として扱います。
同一項目内で複数値を指定した場合、項目内はOR条件として扱います。

例として、担当者に `A` または `B`、ステータスに `Ready` または `InProgress` を指定した場合、以下の条件を満たすタスクを表示します。

- 担当者が `A` または `B`
- かつステータスが `Ready` または `InProgress`

フィルタにより非表示になったタスクに接続する依存関係線は表示しません。

### 並び替え

初期実装で対応する並び替えキーは以下です。

| キー | 用途 |
| --- | --- |
| 依存順 | タスク間接続に基づく表示順 |
| タスク名 | タスク名による表示順 |
| 開始日時 | 予定開始日時による表示順 |
| 終了日時 | 予定終了日時による表示順 |

デフォルトの並び替えは依存順とします。

### UI配置

フィルタ・並び替えUIは、ガントチャート上部に配置します。

ガントチャートの表示内容に直接作用する操作であるため、共通サイドバーではなく、ガントチャート領域内の操作として扱います。

担当者と作業協力者のフィルタは、候補メンバーをプルダウンから追加し、追加済み条件のみを表示します。
チームメンバーが増えた場合でも、全メンバーを常時表示しないUIとします。

### 表示状態の保存

フィルタ・並び替え状態は、アプリケーション設定として `app-settings.json` に保存します。

プロジェクト保存データからは分離し、同じプロジェクトファイルを開いた場合でも、アプリケーション側の表示設定として復元します。

## プロジェクトスケジュール

プロジェクトスケジュールは、1つのプロジェクト内のタスクを表示するガントチャートです。

配置は、プロジェクト内のタスク編集画面にある中央編集領域タブの1つとします。
同じ編集画面には、タスクノードと接続線を編集する「依存関係編集」タブも配置します。
タスク詳細、タスク操作、履歴操作は両タブで共通のサイドバーを使用します。

主な仕様は以下です。

- 同一プロジェクト内のタスクを表示する
- 依存関係編集のタスクノード情報を参照する
- タスク間接続を依存関係として表示する
- タスク名、担当者、開始日時、終了日時を表示する
- ガントチャート上で選択したタスクを共通サイドバーへ反映する
- タスク詳細の変更をガントチャートへ反映する
- ガントチャート上のタスクから依存関係編集上の該当ノードへジャンプできる
- 依存関係編集上のノードからガントチャート上の該当タスクへジャンプできる
- 初期表示時は現在日付付近へ横スクロールする

## チームスケジュール

チームスケジュールは、複数プロジェクトのタスクを横断表示するガントチャートです。

配置は、アプリケーション直下で「チーム内プロジェクト」「チーム設定」と並列にする想定です。

主な仕様は以下です。

- すべてのプロジェクトのタスクを同一表示領域に並列表示する
- プロジェクトを跨いだ依存関係解決は行わない
- プロジェクト別フィルタを検討する
- 担当者別フィルタを検討する
- 作業協力者別フィルタを検討する
- 並び替えを検討する
- プロジェクトスケジュールと同じガントチャート共通資産を利用する

## 予定期間算定

ガントチャート上に表示するタスク予定期間は、タスクの日時情報、作業見積時間、依存関係、担当者の作業可能時間から算定します。

### 基本ルール

| 条件 | 予定期間 |
| --- | --- |
| 開始日時と終了日時が両方設定済み | 開始日時から終了日時まで |
| 開始日時のみ設定済み | 開始日時と作業見積時間から算定 |
| 終了日時のみ設定済み | 作業見積時間から開始位置を逆算 |
| 開始日時と終了日時が未設定 | 前段タスクの最遅終了日時と作業見積時間から算定 |
| 作業見積時間が未設定または `0min` | 描画しない |

開始日時と終了日時が両方設定されている場合、作業見積時間は予定期間算定に使用しません。

### 前段タスク

開始日時と終了日時が未設定のタスクは、Inputに接続されている前段タスクを参照します。

前段タスクが複数ある場合は、予定終了日時が最も遅いタスクの終了日時を基準にします。

前段タスクが存在せず、開始日時も未設定の場合は、現在日付を基準に描画します。

### 担当者の作業可能時間

作業見積時間から予定期間を算定する場合、担当者の作業可能時間を参照します。

参照する情報は以下です。

- メンバーの曜日別デフォルト作業可能時間
- メンバーの特別休日デフォルト作業可能時間
- プロジェクト別参加期間
- プロジェクト別、メンバー別、日付別作業可能時間上書き

日付別作業可能時間が未入力の場合は、曜日別デフォルト作業可能時間を使用します。
対象日が特別休日の場合は、曜日別デフォルトではなく特別休日デフォルト作業可能時間を使用します。

作業可能時間が `0min` の日は、実質的な中断日として扱います。
ガントチャート上では、予定期間と作業不可日が重なる区間に作業不可日表示色を重ねて表示します。

### プロジェクト参加期間

担当者がプロジェクト参加期間外の日付では、そのプロジェクトの作業可能時間を持たないものとして扱います。

参加期間外に予定期間がかかる場合は、タスクバー上の対象区間に参加期間外表示色を重ねて表示します。
表示対象は、予定期間と参加期間外日が重なる区間のみとします。

## 中断期間

タスクには任意数の中断期間を設定できます。

ガントチャートでは、タスクの予定期間と中断期間が重なる場合に、中断区間を視覚的に表示します。

主な仕様は以下です。

- タスク予定期間と重なる中断期間のみ表示する
- タスク予定期間と重ならない中断期間は表示しない
- 重複または連続する中断期間は、算定前に正規化する
- 中断期間はグレーアウトした線または区間として表示する
- 中断期間はガントチャート上で編集可能とする

## 依存関係表示

タスク編集タブ上の接続線は、ガントチャート上ではタスク間依存関係として表示します。

主な仕様は以下です。

- OutputからInputへの接続を依存関係として扱う
- 依存元タスクから依存先タスクへ線を表示する
- プロジェクトスケジュールでは同一プロジェクト内の依存関係を表示する
- チームスケジュールではプロジェクト内の依存関係のみ表示する
- プロジェクトを跨いだ依存関係解決は行わない

## 休日表示

ガントチャートでは、土曜日、日曜日、特別休日を視覚的に識別できるようにします。

休日表示は、プロジェクトスケジュールとチームスケジュールの両方に同じ仕様で適用します。

### 対象日

休日表示の対象日は以下です。

- 土曜日
- 日曜日
- 特別休日

土曜日と日曜日の判定はローカル日付を基準にします。

特別休日は、別途設定された日付を休日として扱います。祝日など、土日以外に作業不可または休日扱いしたい日を登録する用途を想定します。

### 表示方法

休日は、既存のガントチャート背景色に薄いオーバーレイを重ねて表示します。

表示対象は以下です。

- 日付ヘッダーの日付文字列
- チャート本体の縦列全体

左側のタスク名列、担当者列には休日背景色を適用しません。

休日背景色は、タスクバーや依存関係線の視認性を優先し、Light/Darkそれぞれで平日と見分けられる程度の薄い色とします。

### 表示優先順位

休日表示と他の期間表示が重なる場合、背景表示の優先順位は以下です。

1. 現在日
2. 参加期間外
3. 作業不可日
4. 週末または特別休日

背景表示が上位の表示で上書きされる場合でも、日付文字列は休日用文字色を維持します。
これにより、背景表示が重なっても平日と休日を判別できるようにします。

### 特別休日設定

特別休日は、チーム設定から開く別ウィンドウで設定します。

特別休日設定ウィンドウでは、休日設定用カレンダーを表示します。
日付をクリックすると、対象日を特別休日として設定または解除できるようにします。

## 編集

プロジェクトスケジュールでは、ガントチャート本体を直接編集画面として扱わず、タスク選択と状況確認を主な役割とします。
タスク期間や中断期間の編集は、依存関係編集と共通のサイドバーから行います。

主な仕様は以下です。

- ガントチャート上の行またはタスクバーをクリックすると、該当タスクを選択する
- 選択中タスクの詳細情報は共通サイドバーに表示する
- 開始日時、終了日時、作業見積時間、中断期間は共通サイドバーから編集する
- 共通サイドバーからの編集はUndo/Redo対象とする
- タスク詳細側の変更はガントチャートへ反映する

タスクバーの直接ドラッグによる期間編集は、実績日時やステータス算出との関係を整理した上で、必要に応じて再検討します。
現時点では、タスクバー操作で開始日時または終了日時を直接確定しません。

## ステータス表示

タスクステータスはタスク編集機能と同じ自動算出結果を使用します。

対象ステータスは以下です。

- `Ready`
- `Pending`
- `InProgress`
- `Done`

ガントチャート上では、ノードのステータス色と同じ色で表現します。

ステータスはフィルタ条件として利用することも検討します。

## 警告表示

ガントチャート上で警告が必要なタスクは、視覚的に識別できるようにします。

現時点の警告表示方針は以下です。

| 条件 | 表示 |
| --- | --- |
| 終了日時のみ設定されている | タスク名を警告色にする |
| 参加期間外に予定期間がかかる | タスクバー上の対象区間に参加期間外表示色を重ねる |
| 作業不可日に予定期間がかかる | タスクバー上の対象区間に作業不可日表示色を重ねる |

## テーマ

ガントチャートはテーマ設定に対応します。

休日表示では、以下のテーマ項目を追加します。

- 土曜日背景色
- 土曜日文字色
- 日曜日背景色
- 日曜日文字色
- 特別休日背景色
- 特別休日文字色
- 現在日背景色
- 現在日文字色
- 作業不可日表示色
- 参加期間外表示色

その他の追加候補のテーマ項目は以下です。

- ガントチャート背景色
- 時間軸色
- グリッド線色
- タスクバー通常色
- タスクバー選択色
- タスクバー文字色
- プロジェクト区切り色
- 中断期間表示色
- 依存関係線色
- 警告表示色

## 保存データ

初期実装では、ガントチャートはタスク情報とメンバー情報から再生成できる表示機能として扱います。
そのため、現時点ではガントチャート表示状態を保存しません。

ビューのカスタマイズ機能などが追加された場合は、`app-settings.json` への保存を検討します。

将来的な保存候補は以下です。

- 表示開始日
- 表示終了日
- ズーム状態
- スクロール位置
- フィルタ状態
- 並び替え状態
- 折りたたみ状態
