# ステータスバー仕様

このドキュメントは、WcmScheduler のステータスバー機能仕様を整理するための資料です。

ステータスバーは、アプリケーション全体の状態、保存状態、処理中タスク、進捗をユーザーへ通知するための共通UIです。
特に、ガントチャート更新など時間がかかる可能性のある処理の状態確認と、性能改善時の切り分けに利用できることを目的とします。

ここに記載する仕様は、実装状況に応じて更新します。

## 概要

ステータスバーは、メインウィンドウの最下部に常時表示します。

主な役割は以下です。

- 現在の処理状態を表示する
- 処理中の進捗を表示する
- 保存ファイルのダーティ状態を表示する
- 最後に発生した軽微な通知を表示する
- 長時間処理が発生しているかどうかをユーザーが確認できるようにする

## 配置

ステータスバーは、メインウィンドウ全体の最下部に配置します。

以下の画面に対して共通で表示します。

- チーム内プロジェクト
- チーム設定
- 将来追加予定のチームスケジュール

ステータスバーは、各画面のコンテンツ領域ではなく、メインウィンドウの共通領域として扱います。

## 表示項目

初期実装では以下の項目を表示します。

| 項目 | 表示内容 |
| --- | --- |
| 状態メッセージ | 現在の処理状態または最後の通知 |
| 進捗表示 | 処理中の場合にプログレスバーを表示 |
| ダーティ状態 | 未保存変更がある場合に `未保存` を表示 |
| 保存ファイル名 | 現在開いている保存ファイル名 |
| 処理時間 | 直近処理または実行中処理の経過時間 |
| キュー進捗 | 処理キューがある場合に処理済み数と全キュー数 |
| プロジェクト進捗 | プロジェクトごとのステータス別タスク数と進捗率 |

表示領域が不足する場合は、状態メッセージを優先して表示します。

## 状態メッセージ

状態メッセージは、アプリケーションの現在状態または直近の処理結果を表示します。

代表的な表示例は以下です。

| 状態 | メッセージ例 |
| --- | --- |
| 通常時 | `準備完了` |
| 保存中 | `保存中...` |
| 保存完了 | `保存しました` |
| 読み込み中 | `読み込み中...` |
| ガントチャート更新中 | `プロジェクトスケジュールを更新中...` |
| エラー発生時 | `処理に失敗しました` |

処理完了メッセージは時間経過では自動消去しません。
ユーザーがタブを切り替えるなど、操作対象が切り替わったタイミングで通常状態へ戻します。

## 進捗表示

進捗表示は、処理中であることをユーザーへ示すために使用します。

初期実装では、処理の詳細な進捗率が不明なケースを想定し、不定進捗表示を基本とします。

将来的に進捗率を算出できる処理では、決定進捗表示に切り替えます。

| 種別 | 用途 |
| --- | --- |
| 不定進捗 | ガントチャート再計算、保存、読み込みなど進捗率が不明な処理 |
| 決定進捗 | 複数ファイル処理など進捗率を算出できる処理 |

## 処理状態

ステータスバーは、処理中かどうかを示す状態を持ちます。

想定する主な状態は以下です。

| 状態 | 説明 |
| --- | --- |
| Idle | 処理していない状態 |
| Busy | 処理中 |
| Warning | 注意が必要な状態 |
| Error | エラー発生状態 |

複数処理が同時に発生する場合は、表示キューで管理します。
ただし、エラー状態は通常メッセージより優先します。

## 処理キュー

ステータスバーは、処理状態表示用のキューを持ちます。

処理キューが存在する場合、現在処理中または直近処理の状態メッセージに加えて、処理済みキュー数と全キュー数を表示します。

表示例は以下です。

```text
プロジェクトスケジュールを更新中... (1/3)
```

全キュー数は、以下のタイミングでクリアします。

- すべてのキューが実行完了した状態で新規キューが発生した時
- 通常状態へ戻った時

キューに積む対象は、ユーザーが待機状態を認識する必要がある処理とします。

代表例は以下です。

- 保存
- 読み込み
- ガントチャート更新
- 将来追加予定のスケジュール再計算

## 処理時間

ステータスバーには、処理時間の計測値を常時表示します。

処理中は開始からの経過時間を表示します。
処理完了後は、直近処理の所要時間を表示します。

表示例は以下です。

```text
0.8s
```

処理時間は、性能問題の切り分けと通常利用時の待機感の把握に使用します。
デバッグ専用表示にはせず、通常UIとして常時表示します。

## ダーティ状態表示

保存ファイルに未保存変更がある場合、ステータスバーに `未保存` を表示します。

ダーティ判定の仕様は以下を参照します。

- [保存データ仕様](save_data.md)

ダーティ状態は、ウィンドウタイトルやタブ名の `*` 表示と整合する必要があります。

## プロジェクト進捗表示

ステータスバーの右側には、プロジェクトごとのステータス別タスク数と進捗率を表示します。

進捗率は、`Done` のタスク数を全タスク数で割って算出します。

```text
Done 12 / Total 30 (40%)
```

ステータス別タスク数は、以下の分類で表示します。

- Ready
- Pending
- InProgress
- Done

表示領域に余裕がある場合は、すべてのステータス別件数を表示します。
表示領域が不足する場合は、進捗率と `Done / Total` を優先します。

プロジェクトが未選択の場合、またはプロジェクト外の画面では、表示中の操作対象に応じて非表示または全体集計表示とします。

## ガントチャート更新との関係

ガントチャート更新処理では、以下のタイミングでステータスバーを更新します。

| タイミング | 表示 |
| --- | --- |
| 更新開始 | `プロジェクトスケジュールを更新中...` |
| 更新中 | 不定進捗表示 |
| 更新完了 | `プロジェクトスケジュールを更新しました` |
| 更新失敗 | `プロジェクトスケジュール更新に失敗しました` |

ガントチャート更新処理の性能改善を進める際は、更新開始から完了までの時間を把握できるようにすることを検討します。

## エラー表示

ステータスバーは軽微なエラー通知に使用します。

ただし、ユーザー判断が必要なエラーはダイアログを表示します。

| エラー種別 | 表示方法 |
| --- | --- |
| 軽微な通知 | ステータスバー |
| 保存失敗 | ダイアログとステータスバー |
| 読み込み失敗 | ダイアログとステータスバー |
| 復旧判断が必要な状態 | ダイアログ |

## テーマ対応

ステータスバーはテーマ色に対応します。

初期実装で追加するテーマ項目は以下です。

| 項目 | 用途 |
| --- | --- |
| `status-bar-background` | ステータスバー背景色 |
| `status-bar-text` | 通常文字色 |
| `status-bar-border` | 上境界線 |
| `status-bar-warning-text` | 警告文字色 |
| `status-bar-error-text` | エラー文字色 |
| `status-bar-progress-background` | プログレスバー背景色 |
| `status-bar-progress-foreground` | プログレスバー前景色 |

## ViewModel構成

初期実装では、ステータスバーの状態を管理する `StatusBarViewModel` を追加します。

想定する主なプロパティは以下です。

| プロパティ | 役割 |
| --- | --- |
| `Message` | 状態メッセージ |
| `IsBusy` | 処理中かどうか |
| `ProgressValue` | 決定進捗の値 |
| `IsIndeterminate` | 不定進捗かどうか |
| `HasUnsavedChanges` | 未保存変更があるかどうか |
| `CurrentFileName` | 現在の保存ファイル名 |
| `State` | Idle / Busy / Warning / Error |

ステータスバーの状態更新は、各機能のViewModelから直接UIを操作せず、共通の状態管理ViewModelまたはサービスを経由して行います。
