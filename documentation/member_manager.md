# メンバー管理仕様

このドキュメントは、WcmScheduler のチームメンバー管理機能について、公開仕様として整理したものです。
タスク担当者、作業協力者、プロジェクトスケジュール、チームスケジュールで共通して参照するメンバー情報を扱います。

## 概要

メンバー管理は、チームに所属するメンバーを登録し、タスク詳細やスケジュール算定で参照できるようにする機能です。

主な責務は以下です。

- チームメンバーの追加、削除、名称編集
- タスク担当者と作業協力者の選択肢提供
- プロジェクト単位のメンバー参加期間管理
- プロジェクト単位、メンバー単位、日付単位の作業可能時間管理
- プロジェクトスケジュール、チームスケジュールでの担当者別表示や算定への連携

## 配置

メンバー管理UIは、アプリケーション直下のタブとして配置します。

- 「チーム内プロジェクト」タブと並列に「チーム設定」タブを配置します。
- 「チーム設定」タブでメンバー一覧、プロジェクト参加期間、日次作業可能時間を編集します。
- 日次作業可能時間は、選択中メンバーに対する7日分のカレンダーとして表示します。

## メンバー情報

メンバーはチーム内で一意なIDを持ちます。
タスク担当者と作業協力者は、表示名ではなくメンバーIDで保存します。

メンバー情報は以下です。

| 項目 | 内容 |
| --- | --- |
| `MemberId` | メンバーを一意に識別するID。保存データ上の参照キーとして使用します。 |
| `PrimaryName` | メンバー名の主名。 |
| `SecondaryName` | メンバー名の副名。 |
| `SundayWorkTimeMinutes` | 日曜日のデフォルト作業可能時間。単位は分です。 |
| `MondayWorkTimeMinutes` | 月曜日のデフォルト作業可能時間。単位は分です。 |
| `TuesdayWorkTimeMinutes` | 火曜日のデフォルト作業可能時間。単位は分です。 |
| `WednesdayWorkTimeMinutes` | 水曜日のデフォルト作業可能時間。単位は分です。 |
| `ThursdayWorkTimeMinutes` | 木曜日のデフォルト作業可能時間。単位は分です。 |
| `FridayWorkTimeMinutes` | 金曜日のデフォルト作業可能時間。単位は分です。 |
| `SaturdayWorkTimeMinutes` | 土曜日のデフォルト作業可能時間。単位は分です。 |
| `SpecialHolidayWorkTimeMinutes` | 特別休日のデフォルト作業可能時間。単位は分です。 |

UI上の表示名は、`PrimaryName` と `SecondaryName` から都度生成します。
両方が設定されている場合は、`PrimaryName SecondaryName` の形式で表示します。
片方のみ設定されている場合は、設定されている名前のみを表示します。

担当者バッジ用のイニシャルは、`PrimaryName` と `SecondaryName` の先頭1文字から自動生成します。
片方のみ設定されている場合は、設定されている名前の先頭1文字を使用します。
文字セットによる特別扱いは行わず、文字列の先頭1文字をそのまま使用します。

メンバー一覧の表示順は、保存データ内の配列順で保持します。

## プロジェクト参加期間

メンバーの参加期間は、チーム全体ではなくプロジェクト単位で管理します。
同じメンバーでも、プロジェクトごとに異なる参加開始日と参加終了日を設定できます。

プロジェクト参加期間は以下です。

| 項目 | 内容 |
| --- | --- |
| `ProjectId` | 対象プロジェクトID。 |
| `MemberId` | 対象メンバーID。 |
| `ParticipationStartDate` | プロジェクト参加開始日。未入力の場合は開始制限なしとして扱います。 |
| `ParticipationEndDate` | プロジェクト参加終了日。未入力の場合は終了制限なしとして扱います。 |

作業可能時間カレンダーでは、対象日がプロジェクト参加期間内の場合のみ、そのプロジェクトの作業可能時間を入力できます。
参加期間外の日付では作業可能時間入力を無効にします。
参加開始日と参加終了日に矛盾がある場合は、開始日が終了日を超えないように入力時点で補正します。

## タスク担当者と作業協力者

タスク詳細では、担当者と作業協力者をメンバー管理で登録されたメンバーから選択します。

- 担当者はタスク遂行責任を持つメンバーであり、1人のみ設定できます。
- 作業協力者はタスクに関わる補助メンバーであり、複数人を設定できます。
- 未担当状態は担当者IDを `null` として扱います。
- 担当者と作業協力者には、同じメンバーを重複して設定できません。
- UI表示では、保存された `MemberId` からメンバーの表示名を解決します。
- 作業協力者はガントチャート表示上のフィルタリング用タグとして扱います。
- 作業見積とスケジュール算定は担当者を基準に行い、作業協力者の作業可能時間は算定に使用しません。

作業協力者は、タスク詳細で「追加」ボタンを押すことで選択行を追加します。
各選択行ではコンボボックスからメンバーを1人選択します。

## 削除

メンバー削除は、タスクから参照されているかどうかによって扱いを分けます。

- どのタスクからも参照されていないメンバーは物理削除できます。
- タスクから参照されているメンバーは物理削除できません。
- メンバーを削除した場合、そのメンバーに紐づくプロジェクト参加期間と作業可能時間の上書き値も削除します。

## 日次作業可能時間

作業可能時間は、ガントチャートの自動スケジュール算定で利用します。
総作業時間ではなく、プロジェクト単位、メンバー単位、日付単位の日次作業可能時間として扱います。

- 入力単位は分です。
- 各メンバーは曜日ごとのデフォルト作業可能時間を持ちます。
- 各メンバーは特別休日のデフォルト作業可能時間を持ちます。
- カレンダー上で未入力の日付は、対象曜日のデフォルト作業可能時間を採用します。
- 対象日が特別休日の場合、未入力の日付は特別休日のデフォルト作業可能時間を採用します。
- 作業可能時間に0分を入力した日付は、ガントチャート算定上は作業できない日として扱います。
- チーム共通値、チーム全体の合計値などは、日次作業可能時間から算出したViewとして扱います。
- 作業可能時間の入力欄には分単位の値と単位を表示し、補助表示として時間・分表記も併記します。
- 日付別の作業可能時間が未入力の場合は、入力欄内に対象曜日または特別休日のデフォルト値をplaceholderとして表示します。

作業可能時間カレンダーは、1ページあたり7日分を表示します。
日付の並びは、常に日、月、火、水、木、金、土の順とします。
前後移動は7日単位で行います。

## 特別休日

特別休日は、土曜日または日曜日とは別に休日扱いする日付です。
祝日など、チーム全体で共通して休日扱いしたい日付を登録します。

特別休日は、チーム設定タブから開く特別休日設定ウィンドウで編集します。
特別休日設定ウィンドウでは、カレンダーの日付クリックによって特別休日を設定または解除します。

特別休日に設定された日付は、ガントチャートの休日表示とスケジュール算定で参照します。
メンバーごとの特別休日デフォルト作業可能時間が `0min` の場合、そのメンバーは特別休日に作業できないものとして扱います。

## 保存データ

メンバー情報はチーム全体の保存データに保持します。
JSONキーは kebab-case とします。

```json
{
  "members": [
    {
      "member-id": "00000000-0000-0000-0000-000000000000",
      "primary-name": "Alex",
      "secondary-name": "",
      "sunday-work-time-minutes": 0,
      "monday-work-time-minutes": 480,
      "tuesday-work-time-minutes": 480,
      "wednesday-work-time-minutes": 420,
      "thursday-work-time-minutes": 480,
      "friday-work-time-minutes": 480,
      "saturday-work-time-minutes": 0,
      "special-holiday-work-time-minutes": 0
    }
  ],
  "special-holidays": [
    "2026-05-04"
  ]
}
```

プロジェクト参加期間は、プロジェクトIDとメンバーIDの組み合わせで保存します。

```json
{
  "project-member-participations": [
    {
      "project-id": "00000000-0000-0000-0000-000000000000",
      "member-id": "00000000-0000-0000-0000-000000000000",
      "participation-start-date": "2026-04-01",
      "participation-end-date": null
    }
  ]
}
```

プロジェクト単位のメンバー別作業可能時間は、日付別の上書き値のみを保存データに保持します。

```json
{
  "project-member-work-times": [
    {
      "project-id": "00000000-0000-0000-0000-000000000000",
      "member-id": "00000000-0000-0000-0000-000000000000",
      "work-date": "2026-04-27",
      "work-time-minutes": 480
    }
  ]
}
```

タスク詳細側では担当者IDと作業協力者IDを保持します。

```json
{
  "assignee-member-id": "00000000-0000-0000-0000-000000000000",
  "collaborator-member-ids": [
    "00000000-0000-0000-0000-000000000001"
  ]
}
```

## UI仕様

### チーム設定タブ

チーム設定タブでは、メンバー一覧、プロジェクト参加期間、作業可能時間カレンダーを同じ画面で編集します。

主な操作は以下です。

- メンバー追加
- メンバー名変更
- メンバー削除
- メンバーの並び替え
- 曜日ごとのデフォルト作業可能時間の入力
- プロジェクト単位の参加開始日、参加終了日の入力
- プロジェクト単位、メンバー単位、日付単位の作業可能時間編集

### タスク詳細

タスク詳細では、担当者と作業協力者をメンバー一覧から選択します。

- 未担当を選択できます。
- 担当者は単一選択とします。
- 作業協力者は複数選択とします。
- 担当者の保存値は `MemberId` とします。
- 作業協力者の保存値は `MemberId` の一覧とします。
- 表示名はメンバー管理から解決します。

### 担当者バッジ

担当者バッジは、担当者メンバーを視覚的に識別するための表示です。

- 初期実装では、担当者メンバーのイニシャルをデフォルト表示します。
- イニシャルは自動生成のみとします。
- 将来的には、ユーザーが設定した画像をバッジ表示に使用できるようにする想定です。
- バッジ色はテーマ色を使用します。
- タスクノードでは右上に表示します。
- ガントチャートでも同じバッジを表示します。
